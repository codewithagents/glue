`@codewithagents/openapi-react-query` generates typed [React Query v5](https://tanstack.com/query/v5) hooks from your OpenAPI spec. One `useQuery` hook per GET endpoint, one `useMutation` hook per write operation. No hand-written boilerplate.

## Overview

- One hook per operation: `useQuery` for GET, `useMutation` for POST/PUT/PATCH/DELETE.
- Types derived directly from the generated client. No duplication between spec, client, and hooks.
- Key factories included per resource: `all()`, `list(params)`, `detail(id)`.
- Smart detail hooks: path-param hooks disable automatically when the param is `null` or `undefined`. No `enabled: !!id` at every call site.
- Auto-invalidate on mutation: set `auto_invalidate: true` to invalidate related queries on mutation success.
- Suspense variants: set `suspense: true` to generate `useSuspense*` hooks alongside every query hook.

## Install

```bash
npm install -D @codewithagents/openapi-react-query @codewithagents/openapi-gen
npm install @tanstack/react-query
```

## Minimal example

**Config (`openapi-react-query.config.json`):**

```json
{
  "input_openapi": "./openapi.json",
  "output": "./src/api"
}
```

**Run both generators:**

```bash
npx openapi-gen
npx openapi-react-query
```

**Use the generated hooks:**

```tsx
import { usePetDetail } from './src/api/hooks';

function PetCard({ petId }: { petId: number }) {
  const { data, isLoading } = usePetDetail(petId);
  if (isLoading) return <p>Loading...</p>;
  return <h2>{data?.name}</h2>;
}
```

## Configuration

| Field | Type | Required | Description |
|---|---|---|---|
| `input_openapi` | `string` | Yes | Path to your OpenAPI spec (JSON or YAML) |
| `output` | `string` | Yes | Directory to write generated files (same as openapi-gen output) |
| `stale_time` | `number` | No | Global `staleTime` in milliseconds |
| `gc_time` | `number` | No | Global `gcTime` in milliseconds |
| `auto_invalidate` | `boolean` | No | Invalidate related queries on mutation success |
| `suspense` | `boolean` | No | Generate `useSuspense*` variants |

## Generated output

The generator writes a `hooks.ts` file to your output directory alongside the client generated by `openapi-gen`.
